Below is a **simple, clear, step-by-step guide** to help you understand **Continuous Integration (CI)** by **installing, configuring, and setting up Jenkins** on a **Linux system**.

This guide assumes **Ubuntu/Debian**, but steps are similar for CentOS/RHEL.

---

# âœ… **What is Jenkins? (Simple Explanation)**

Jenkins is an **automation server** used for:

* Continuous Integration (CI)
* Continuous Delivery (CD)
* Automating build, test, and deployment processes

It helps developers automatically build and test code whenever they push changes.

# ğŸŸ¢ **1. Install Jenkins on Linux (Ubuntu/Debian)**

### **Step 1: Update packages**

```
sudo apt update
```

---

### **Step 2: Install Java (required for Jenkins)**

Jenkins runs on Java.

```
sudo apt install openjdk-11-jdk -y
```

Check version:

```
java -version
```

---

### **Step 3: Add Jenkins repository**

```
curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io.key | sudo tee \
  /usr/share/keyrings/jenkins-keyring.asc > /dev/null
```

Add Jenkins repository file:

```
echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \
  https://pkg.jenkins.io/debian-stable binary/ | sudo tee \
  /etc/apt/sources.list.d/jenkins.list > /dev/null
```

---

### **Step 4: Install Jenkins**

```
sudo apt update
sudo apt install jenkins -y
```

---

### **Step 5: Start and Enable Jenkins**

```
sudo systemctl start jenkins
sudo systemctl enable jenkins
```

Check status:

```
systemctl status jenkins
```

Jenkins is now running ğŸ‰
